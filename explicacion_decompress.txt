EXPLICACION DE DECOMPRESS.C
(como si tuvieras 10 años)
============================

Este archivo es el DESCOMPRESOR. Su trabajo es lo OPUESTO al compresor:
  Recibir el archivo .bin que creo compress
  -> reconstruir todos los archivos de texto originales en una carpeta

Es como abrir la mochila magica y sacar todos los libros tal como eran.


================================================
LA HISTORIA COMPLETA (lo que hace main)
================================================

Cuando ejecutas el programa asi:
  ./decompress resultado.bin carpeta_salida

El programa hace esto en orden:

  1. Revisa que le diste exactamente 2 argumentos.
     Si no, te explica como usarlo y se cierra.

  2. Intenta crear la carpeta de salida.
     Si ya existe, no pasa nada (no es un error).

  3. Abre el archivo .bin para leerlo.

  4. Lee los primeros 4 bytes y verifica que digan "HUF1".
     Si no dicen eso, el archivo no es valido y se cierra.
     (Como verificar que la etiqueta de la mochila sea la correcta)

  5. Lee cuantos archivos hay adentro.

  6. Por cada archivo, llama a decode_file() para recuperarlo.

  7. Cierra el .bin y te dice "listo!".


================================================
FUNCION: decode_file(in, output_dir)
================================================

Esta es la unica funcion importante del archivo. Recupera UN archivo
del .bin y lo escribe en disco.

Recuerda que compress guardo esta info por cada archivo:
  [largo ruta][ruta relativa][tamaño original][freq x256][bits comprimidos]

decode_file lee exactamente eso, en el mismo orden.

--- PASO 1: Leer la cabecera ---

  Lee el largo de la ruta (2 bytes).
  Lee la ruta relativa (ej: "docs/notas.txt").
  Lee el tamaño original del archivo (cuantos bytes tenia).
  Lee la tabla de frecuencias (256 numeros de 8 bytes cada uno).

--- PASO 2: Crear las carpetas necesarias ---

  Si la ruta es "docs/sub/notas.txt", primero tiene que existir
  la carpeta "docs/sub/" en la carpeta de salida.
  Llama a ensure_parent_dirs() para crearlas si no existen.

--- PASO 3: Abrir el archivo de salida ---

  Crea el archivo vacio donde va a escribir el contenido recuperado.
  Ej: carpeta_salida/docs/sub/notas.txt

--- CASO ESPECIAL: archivo vacio ---

  Si original_size == 0, el archivo original estaba vacio.
  No hay nada que descomprimir, cierra el archivo y listo.

--- PASO 4: Reconstruir el arbol de Huffman ---

  Con la tabla de frecuencias que leimos, llama a build_huffman_tree().
  Esto reconstruye EXACTAMENTE el mismo arbol que uso el compresor.
  (porque usamos las mismas frecuencias -> mismo arbol -> mismos codigos)

--- CASO ESPECIAL: archivo con un solo tipo de byte ---

  Si el arbol tiene solo un hijo izquierdo (un solo caracter unico),
  es un caso raro donde solo habia un byte posible en el archivo.
  En ese caso simplemente escribe ese mismo byte original_size veces.
  Tambien consume los bits de relleno del buffer.

--- PASO 5: Descomprimir leyendo bit a bit ---

  Aqui esta la magia. Para recuperar cada byte original:

    Empieza en la RAIZ del arbol.
    Lee un bit del archivo comprimido:
      Si es 0 -> ve al hijo IZQUIERDO
      Si es 1 -> ve al hijo DERECHO
    Repite hasta llegar a una HOJA (is_leaf == 1).
    La hoja te dice que letra era. Escribela al archivo de salida.
    Vuelve a la raiz y repite para el siguiente byte.

  Hace esto exactamente original_size veces (hasta recuperar todos
  los bytes que tenia el archivo original).

EJEMPLO:

  El arbol era:
          [5]
         /   \
       [3]    c=8
      /   \
    a=2   b=3

  Los codigos eran: a=00, b=01, c=1

  El archivo comprimido tiene los bits: 00 00 01 01 1
                                         a   a  b  b  c

  Descompresion:
    Bit 0 -> izquierda -> [3]
    Bit 0 -> izquierda -> hoja a -> escribe 'a'

    Bit 0 -> izquierda -> [3]
    Bit 0 -> izquierda -> hoja a -> escribe 'a'

    Bit 0 -> izquierda -> [3]
    Bit 1 -> derecha   -> hoja b -> escribe 'b'

    Bit 0 -> izquierda -> [3]
    Bit 1 -> derecha   -> hoja b -> escribe 'b'

    Bit 1 -> derecha -> hoja c -> escribe 'c'

  Resultado: "aabbc"  <- exactamente el original!


================================================
POR QUE FUNCIONA?
================================================

El truco es que el descompresor puede reconstruir el MISMO arbol
porque el compresor guardo la tabla de frecuencias en el .bin.

Con las mismas frecuencias -> mismo arbol -> mismos codigos.
Entonces al leer los bits comprimidos y recorrer el arbol, siempre
llegamos a la letra correcta.

Es como tener un mapa secreto: el compresor te lo guarda adentro del
archivo comprimido para que el descompresor pueda usarlo.


================================================
ORDEN EN QUE SE LLAMAN LAS FUNCIONES
================================================

main()
  |
  +-> MKDIR() crea carpeta de salida
  +-> fopen() abre el .bin
  +-> fread() verifica la firma "HUF1"
  +-> read_u32() lee cuantos archivos hay
  |
  +-> [para cada archivo]:
        decode_file()
          +-> read_u16()          <- lee largo de ruta
          +-> fread()             <- lee la ruta relativa
          +-> read_u64()          <- lee tamaño original
          +-> read_u64() x256     <- lee tabla de frecuencias
          +-> ensure_parent_dirs()<- crea carpetas necesarias
          +-> fopen()             <- crea el archivo de salida
          +-> build_huffman_tree()<- reconstruye el arbol
          +-> bit_reader_init()   <- prepara el lector de bits
          +-> bit_reader_read_bit()<- lee bit a bit
          +-> fputc()             <- escribe cada byte recuperado
          +-> free_huffman_tree() <- libera la memoria
          +-> fclose()            <- cierra el archivo recuperado
  |
  +-> fclose() el .bin
  +-> imprime "Descompresion completada!"

================================================
COMO COMPILAR Y PROBAR EL PROYECTO
================================================
================================================

No hay Makefile, asi que compilas a mano con gcc.
Todos los archivos estan en la carpeta serial/.

--- PASO 1: Entrar a la carpeta ---

  cd serial

--- PASO 2: Compilar el compresor ---

  gcc -Wall -o compress compress.c huffman.c

--- PASO 3: Compilar el descompresor ---

  gcc -Wall -o decompress decompress.c huffman.c

--- PASO 4: Crear carpeta de prueba con archivos de texto ---

  mkdir -p prueba/subcarpeta
  echo "hola mundo, esto es una prueba de huffman!" > prueba/hola.txt
  echo "otro archivo de texto con mas contenido repetido repetido" > prueba/otro.txt
  echo "archivo en subcarpeta" > prueba/subcarpeta/sub.txt

--- PASO 5: Comprimir ---

  ./compress prueba resultado.bin

  Deberia imprimir algo como:
    Comprimido: hola.txt
    Comprimido: otro.txt
    Comprimido: subcarpeta/sub.txt
    Archivo generado: resultado.bin

--- PASO 6: Descomprimir en otra carpeta ---

  ./decompress resultado.bin recuperado

  Deberia imprimir:
    Recuperado: hola.txt
    Recuperado: otro.txt
    Recuperado: subcarpeta/sub.txt
    Descompresion completada en: recuperado

--- PASO 7: Verificar que los archivos son identicos ---

  diff prueba/hola.txt recuperado/hola.txt
  diff prueba/otro.txt recuperado/otro.txt
  diff prueba/subcarpeta/sub.txt recuperado/subcarpeta/sub.txt

  Si diff no imprime nada -> los archivos son IDENTICOS. Funciono!

  O para verificar todos de un solo comando:
  diff -r prueba recuperado

--- PASO 8 (opcional): Ver cuanto se comprimio ---

  ls -lh resultado.bin
  du -sh prueba/

  Si resultado.bin pesa menos que la carpeta prueba/ -> comprimio bien!
